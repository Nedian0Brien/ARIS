# 이슈 리포트: 새 세션 모달 UI 깨짐 및 동작 오류 해결

## 1. 개요
새 세션 시작 모달에서 텍스트가 서로 붙어 출력되는 레이아웃 깨짐 현상과, 이후 코드 수정 과정에서 모달이 열리지 않거나 접근성 경고가 발생하는 복합적인 이슈를 해결함.

## 2. 상세 내용

### 이슈 1: 텍스트 뭉침 및 레이아웃 파손
- **증상**: 에이전트 설명 텍스트가 공백 없이 일렬로 붙어 출력됨 (예: `Balanced coding flowFast implementationBroad reasoning`).
- **원인**: 
  - Next.js 15 (App Router) 환경에서 프로덕션 빌드 시 `<style jsx>` 블록의 스타일 정보가 정상적으로 컴파일되지 않고 유실됨.
  - 스타일이 적용되지 않아 `div`, `section` 등 블록 요소들이 기본 인라인 상태처럼 렌더링됨.
  - 특정 환경에서 CSS `gap` 속성이 인식되지 않아 요소 간 간격이 사라짐.
- **해결**:
  - 불안정한 인라인 스타일(`styled-jsx`)을 완전히 제거하고 모든 스타일을 `globals.css`로 이전함.
  - `gap` 대신 `margin` 기반 레이아웃을 사용하여 브라우저 호환성을 확보함.
  - 모든 텍스트 요소를 `display: block`으로 설정하여 스타일 유실 시에도 텍스트 결합을 방지함.

### 이슈 2: 모달 미출력 (열리지 않음)
- **증상**: "새 세션 시작" 버튼을 클릭해도 화면에 변화가 없음.
- **원인**: 
  - 스타일 이전 및 코드 정리 과정에서 모달을 화면에 렌더링하는 `{createModal}` JS 호출 로직이 실수로 삭제됨.
  - 모달 배경과 위치를 정의하는 `.modal-overlay` 클래스가 `globals.css`에서 누락되어 모달이 투명하거나 화면 밖에 렌더링됨.
- **해결**:
  - `SessionDashboard.tsx` 하단에 `{createModal}` 렌더링 로직을 복구함.
  - `globals.css`에 `.modal-overlay` 스타일(position, z-index, background 등)을 다시 정의함.

### 이슈 3: 접근성 및 자동완성 경고
- **증상**: 브라우저 개발자 도구에서 `id` 또는 `name` 속성 누락 및 `label` 연결성 오류 경고 발생.
- **원인**: 폼 입력 필드(`Input`)에 식별자가 없었고, `label`의 `htmlFor` 속성이 누락되어 브라우저 자동완성이 불가능했음.
- **해결**:
  - `Input` 요소에 `id="project-path"`, `name="projectPath"` 추가.
  - 관련 `label`에 `htmlFor="project-path"`를 명시하여 접근성 표준을 준수함.

## 3. 해결 결과 및 검증
- **빌드 성공**: `next build`를 통해 모든 스타일과 코드가 정상 컴파일됨을 확인.
- **배포 완료**: 도커 컨테이너 재빌드 후 실서버에 반영하여 모든 기기(모바일/데스크톱)에서 정상 작동 확인.
- **UI 개선**: 프리텐다드(Pretendard) 기반 폰트 스택을 적용하여 한글/영문 가독성 향상.

## 4. 재발 방지 대책
- 복잡한 모달 UI나 전역적으로 영향을 주는 컴포넌트는 인라인 스타일보다는 외부 CSS 파일(`globals.css` 또는 CSS Modules)을 우선적으로 사용함.
- 코드 리팩토링 시 렌더링 루프 바깥의 리턴문(`{createModal}`) 등 핵심 로직 누락 여부를 철저히 검증함.
- 접근성 경고(id/name/label)를 사전에 방지하기 위해 폼 요소 작성 가이드라인을 준수함.
