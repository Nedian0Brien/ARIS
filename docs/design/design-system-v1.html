for the ARIS console.

고밀도 정보와 여유 있는 공기감. 정제된 블루를 중심으로 한 ARIS 디자인 언어는 에이전트 세션 관리에 필요한 명료함과 속도감을 모두 제공합니다. 모든 토큰은 CSS 변수로 노출되며, 라이트 모드를 기본으로 하고 다크 모드를 동등한 수준으로 지원합니다.

01 · Principles

Five principles that anchor every decision.

디자인 결정은 단일 토큰이 아니라 원칙에서 출발합니다. 새로운 컴포넌트를 만들 때 아래 다섯 가지와 충돌하면 토큰보다 원칙이 우선합니다.

01

Calm surface, confident hierarchy

화면은 조용해야 하고, 위계는 선명해야 합니다. 컬러로 외치지 말고 타이포와 공간으로 말합니다.

02

Monochrome first, accent last

Blue는 액션과 포커스에만 씁니다. 기본 UI는 뉴트럴 스케일로 구성해 색의 의미를 지킵니다.

03

Keyboard is a first-class citizen

모든 주요 액션에는 단축키가 있어야 합니다. ⌘K는 어디서든 같은 뜻입니다.

04

Motion that respects the user

120–180ms · ease-out-expo가 기본. 느린 트랜지션은 오만하며, 없는 트랜지션은 거칩니다.

05

Dark mode is not a filter

다크는 라이트의 반전이 아니라 별도의 팔레트입니다. 동일한 정보 밀도와 대비를 유지합니다.

07 · Foundations — Motion

Snap, don't swoosh.

트랜지션 시간은 120·180·260ms 세 단계. 기본 easing은 cubic-bezier(0.16, 1, 0.3, 1). 카드 위에 커서를 올려보세요.

Fast · 120ms · ease-smooth
hover · color swap · button press
Normal · 180ms · ease-snap
modal enter · tab switch · drawer
Slow · 260ms · ease-spring
progress fill · emphasis · reveal
