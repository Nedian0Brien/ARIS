Desktop anatomy
1

Sessions sidebar · 264px

상단 로고 · New chat · 검색 · 그룹 리스트(Pinned / Today / Yesterday / Earlier) · 하단 유저. 활성 세션은 b-50 배경과 b-500 pulse dot으로 표시합니다.

2

Chat header · 52px

세션명 + running badge + 실행 시간/토큰 meta. 우측은 모델 selector와 share/workspace toggle/more. 모든 정보는 한 줄에 수렴합니다.

3

Timeline · max 780px

읽기 편한 폭으로 중앙 정렬. User 버블은 b-50, Agent는 bubble 없이 본문 그대로. Tool call은 collapsed chip / expanded 카드 두 형태.

4

Jump bar · sticky above composer

마지막 유저 메시지를 잃어버리지 않게 항상 스크롤 맨 아래 부근에 대기. glassmorphism으로 배경을 방해하지 않습니다. 닫으면 세션 동안 숨김 유지.

5

Composer · focus-first

한 장짜리 카드. 상단에 첨부 chip, 중단에 textarea, 하단에 툴/모델/Send. 포커스 시점에 brand 링이 켜져 인풋에만 집중됩니다. ⌘↵으로 전송.

6

Workspace pane · 420px

Run timeline · Files diff · Preview · Terminal 탭. 에이전트가 건드린 파일과 실행 단계를 한눈에. ⌘⇧\\ 으로 토글, 숨기면 채팅이 더 넓어집니다.

02 · Mobile · 390px

Single column, full attention.

모바일에서는 한 번에 하나씩. 사이드바와 워크스페이스는 각각 좌측 드로어와 하단 시트로 밀려 들어갑니다. 인풋은 키보드와 함께 올라오고, 점프바는 스크롤 복원 후에도 방해되지 않는 위치에 뜹니다.

06 · Rationale

다섯 가지 결정, 한 줄로.

A

왜 User만 버블?

Agent는 긴 본문·코드·리스트를 자주 씁니다. 버블에 갇히면 가독성이 깨집니다. 시각적 구분은 아바타와 들여쓰기로 충분합니다.

B

왜 모바일 헤더에 모델 빼나?

좁은 화면에서 헤더에는 "내가 어디에 있는지"만 남깁니다. 모델 변경은 composer에서 처리하므로 맥락과 붙어있습니다.

C

왜 점프바는 라운드 필?

sticky 요소는 콘텐츠와 분리되어야 인지됩니다. 라운드 필 + blur 배경으로 "떠 있는 오버레이"임을 보여줍니다.

D

왜 워크스페이스가 우측?

좌측 사이드바는 세션 간 이동, 우측 패널은 세션 내부 컨텍스트. 관심 영역을 물리적으로 분리해 인지 비용을 줄입니다.

E

왜 stop 버튼은 danger?

실행 중단은 되돌릴 수 있지만, 에이전트 응답 중간에 끊으면 맥락이 깨집니다. 의도를 확신할 때만 누르도록 빨간색.

F

왜 타임라인 max-width 780px?

한국어 본문 14px 기준, 한 줄 75~85자가 가장 읽기 편합니다. 그 이상은 시선이 줄 끝에서 다음 줄 시작을 놓칩니다.
