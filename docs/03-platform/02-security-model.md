# ARIS Security Model

## 1. 목표

- 인증 없는 접근 차단
- 최소 권한 원칙 적용
- 전송/저장 구간의 기밀성 보장
- 모든 민감 액션 감사로그 기록

## 2. 인증/권한

- 로그인 필수: `/login` 전 런타임 화면 접근 불가
- **2FA (Two-Factor Authentication)**: 등록되지 않은 새로운 기기에서 접근 시 2차 인증(OTP 등) 요구
- **Device Trust**: 신뢰할 수 있는 기기 목록 관리 및 관리자 승인 기반 접근
- 역할:
  - `operator`: 지시 전송, 권한 승인, 세션 제어, SSH fallback
  - `viewer`: 조회 전용
- API/웹 라우트 모두 인증 가드 적용

## 3. 비밀정보 정책

- 이메일/비밀번호/토큰/키는 코드/문서/git에 저장 금지
- 운영 비밀값은 `deploy/.env` 등 gitignored 파일로만 관리
- 노출 사고 발생 시 즉시 회전(rotating) + 기록 정리

## 4. 암호화 정책

- 전송: HTTPS/TLS, WSS 강제
- 저장: 민감 데이터 암호화 및 최소 보존
- 세션 메시지: 가능한 E2E 암호화 경계를 유지

## 5. 감사 로그

필수 수집 이벤트:
- 로그인/로그아웃
- 세션 액션(abort/retry/kill/resume)
- permission 승인/거절
- SSH fallback 발급/진입/종료

## 6. SSH fallback 보안

- 기본 비활성 또는 제한적 활성
- operator 전용
- TTL 기반 단기 접근
- 모든 접근/명령 감사

## 7. 보안 체크리스트

- 인증 우회 경로가 없는가
- 쿠키/세션 만료가 정상 동작하는가
- 민감 이벤트가 누락 없이 감사로그에 기록되는가
- 비밀값이 git에 포함되지 않았는가
