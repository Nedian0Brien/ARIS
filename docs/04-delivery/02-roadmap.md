# ARIS Roadmap

## Phase 0. Intent Lock

- 사용자 의도 기준선 확정
- 문서 구조 재정렬
- 디자인/보안 원칙 고정

## Phase 1. Chat Workspace Core

- 세션 목록 + 대화 스트림 + 컴포저 구현
- 이벤트 타입 렌더링 체계 확정
- 기본 반응형 적용

완료 기준:
- 채팅형 작업 흐름으로 지시/응답 확인 가능

## Phase 2. Actionability

- permission 처리 UI/동작
- 세션 액션(abort/retry/kill/resume)
- 모바일 퀵 액션 개선

완료 기준:
- 웹에서 운영 액션 완료 가능

## Phase 3. Security Hardening

- 로그인/역할 권한 강화
- 감사로그 강화
- 비밀정보 관리/회전 절차 정리

완료 기준:
- 인증 우회 경로 제거
- 민감 이벤트 추적 일관화

## Phase 4. Hosting & Reliability

- docker compose 운영 안정화
- 도메인/HTTPS 운영 점검
- 장애 대응 runbook 정리

완료 기준:
- 운영 환경에서 안정적 접속/복구 가능

## Phase 5. Post-MVP

- Skills 시각화 관리
- 지침 문서 시각화 관리
- 프로젝트별 실행 이력/문서 허브
