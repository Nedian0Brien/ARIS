# Plan: `_legacy` Boundary Refactor

**Generated**: 2026-07-08  
**Estimated Complexity**: High  
**Issue**: https://github.com/Nedian0Brien/ARIS/issues/376

## Overview

리디자인 이후 활성 화면은 `app/HomePageClient.tsx`와 `components/project-chat/**`가 기준이다. 하지만 활성 코드가 아직 `app/_legacy/sessions/[sessionId]/**` 안의 helper, Markdown renderer, scroll orchestration을 직접 import한다. 이 상태에서는 `_legacy`를 격리 폴더라고 부를 수 없고, 삭제 가능 범위도 판단하기 어렵다.

해결 목표는 `_legacy` 전체 삭제가 아니다. 1차 목표는 활성 제품 코드에서 `_legacy` import를 0개로 만드는 것이다. 그 뒤 `_legacy` 안에 남은 파일을 테스트 전용, 비활성 화면 전용, 삭제 후보로 다시 분류한다.

## Current Evidence

- `services/aris-web/app/HomePageClient.tsx:69-75`가 `_legacy/.../chat-screen/helpers`와 `MarkdownContent`를 import한다.
- `services/aris-web/components/project-chat/ProjectChatSurface.tsx:92-98`도 같은 helper와 `MarkdownContent`를 import한다.
- `services/aris-web/components/layout/Header.tsx:11`, `BottomNav.tsx:6`, `ViewportHeightSync.tsx:4-5`, `mobileScrollAutoHide.ts:1`이 `_legacy`의 scroll orchestration/debug/type에 의존한다.
- `MarkdownContent`는 `ChatInterface.module.css`, `ResourceChip`, `CodeHighlighter`, `chatFileReferences`, `helpers`에 연결되어 있어 파일 하나만 옮기면 끝나지 않는다.
- `origin/cleanup/homepageclient-dead-code`는 이름과 달리 backend import/subagent schema와 테스트 삭제까지 포함한다. `#376` 해결에 그대로 쓰면 안 된다.

## Non-goals

- `_legacy` 전체 삭제를 첫 PR에 포함하지 않는다.
- `ProjectChatSurface.tsx`의 대형 파일 분해는 `#362` 범위로 둔다.
- mobile visual redesign, 새 UI shell, 새 interaction model을 만들지 않는다.
- import/subagent 기능, `parentChatId` schema, runtime API 계약을 건드리지 않는다.

## Rollback Plan

- 각 sprint를 별도 commit으로 쪼갠다.
- Markdown renderer 이동 후 시각/렌더 테스트가 흔들리면 Sprint 2만 되돌린다.
- Scroll orchestration 이동 후 모바일 회귀가 보이면 Sprint 3만 되돌리고 helper/Markdown 추출은 유지한다.
- `_legacy` 파일 삭제는 이 계획의 마지막 산출물이 아니다. 삭제는 별도 PR에서 진행한다.

## Risks And Mitigations

- **Markdown CSS drift**: `ChatInterface.module.css`에서 필요한 class만 새 CSS module로 복사하면 누락 위험이 있다. `chatRenderers.test.ts`와 실제 project chat smoke를 같이 본다.
- **Test-only imports**: 많은 테스트가 `_legacy`를 직접 참조한다. 제품 import 0개와 테스트 정리는 분리해서 진행한다.
- **Cleanup branch confusion**: `origin/cleanup/homepageclient-dead-code`는 범위가 너무 넓다. cherry-pick 대신 필요한 diff만 수동 재작성한다.
- **Typecheck blocker**: `#377` 때문에 web 전체 typecheck가 실패한다. 이 작업에서는 targeted test, lint, import boundary test로 검증하고, `#377` 이후 전체 typecheck를 붙인다.
