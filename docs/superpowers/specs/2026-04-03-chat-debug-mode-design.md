# Chat Debug Mode Design

**Goal:** 채팅 상단 헤더에 디버그 모드를 추가하고, 좁은 화면에서는 메뉴 안으로 이동시키며, 활성화 시에는 ARIS의 파싱 결과 대신 에이전트 원문을 보여준다.

**Architecture:** `ChatInterface`가 디버그 모드 상태를 소유한다. 헤더 컨테이너의 실제 폭을 측정해 토글을 헤더에 노출할지 메뉴에 넣을지 결정한다. 디버그 모드에서는 agent action 카드와 요약 렌더링을 우회하고, 원문 `body`를 기존 마크다운/코드 렌더러 또는 쉘 트랜스크립트 렌더러로 출력한다.

**Behavior:** 디버그 모드는 개발용이므로 새로고침 시 항상 꺼진 상태로 시작한다. 토글은 현재 페이지 생명주기 동안만 유지되며 저장하지 않는다. action event는 원문을 읽기 쉽게 표시하고, prose는 기존 마크다운 렌더링을 그대로 유지한다. 쉘 트랜스크립트로 보이는 경우에는 bash syntax highlighting을 적용한다.

**Risks / Tradeoffs:** 원문 판별은 휴리스틱 기반이어서 일부 메시지는 prose처럼, 일부는 transcript처럼 보일 수 있다. 그러나 이 접근은 데이터 모델을 건드리지 않고 현재 UI 구조에 가장 적은 변경으로 요구사항을 만족한다.

**Testing:** 헤더 노출 기준과 transcript 판별 로직을 단위 테스트로 검증하고, 기존 채팅 관련 테스트를 함께 실행한다.
