01 · What changes

에이전트가 화면에 보이게 한다.

v2에서는 에이전트 이름이 텍스트로만 적혀 있었습니다. v3 Agents는 모든 화면에서 에이전트가 색·아바타·칩·노드로 표시되도록 일관된 컴포넌트 셋을 도입합니다.

  • ① Agent chip — 모든 카드·메시지·세션에 [•] Claude 형태의 모노 칩. v2 badge보다 작고 식별성 높음.
  • ② Agent avatar 시스템 — 24·32·40px 3사이즈, 이니셜 1자 + 상태 pip(우하단). 라이트·다크 통일.
  • ③ Agent message card — 좌측 1px 컬러 라인 + 헤더(이름·모델 배지·시간). 멀티 에이전트가 같은 채팅에 답해도 한눈에 구별.
  • ④ Topology strip — 채팅 헤더 가로 줄. 활성 에이전트 노드가 좌→우로 배치, 응답 중인 노드는 살짝 커지고 색이 들어감.
  • ⑤ Multi-agent stack — 한 사용자 질문에 여러 에이전트가 답할 때 묶어서 표시. 비교 ·합의 인터페이스의 기반.
  • ⑥ Composer 모델 선택기 — v2 텍스트 드롭다운 대신 가시적 model chip toggle. on/off가 즉시 보임.
원칙

색은 식별 신호지 장식이 아니다. 한 화면에 동시에 보이는 에이전트는 최대 5개. 그 이상은 중첩 아바타 + N+ 칩으로 축약.

06 · Risk

색은 신호다, 장식이 아니다.

① 한 화면 5개 한도. 동시 표시 에이전트 5개를 넘으면 chip을 중첩 아바타 + +N으로 축약. 표 화면에서는 컬러 강도 25% 감소.

② 색약 보완. 색 단독으로 식별하지 않음 — 항상 이니셜·이름·모델 배지를 동반. 색은 보조 신호.

③ 새 에이전트 추가. Claude/Codex/Gemini 외 모델은 일단 채도 낮은 그레이 스케일로 등록 → 사용 빈도가 일정 이상이면 컬러 토큰 추가 후보.

④ Project 화면. 같은 컴포넌트가 프로젝트 카드의 "이 프로젝트에서 자주 쓰는 에이전트"에도 그대로 적용 — 별도 디자인 없이 재사용.

결론

다른 멀티 에이전트 도구는 모델 이름을 텍스트로만 적습니다. ARIS v3 Agents는 색·아바타·노드·라인을 한 시스템으로 묶어 멀티 에이전트라는 정체성을 첫 화면에서 전달합니다.
