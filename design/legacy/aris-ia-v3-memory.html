01 · What changes

컨텍스트가 보이는 워크스페이스.

ARIS의 진짜 차별점은 같은 프로젝트 안에 AGENTS.md · MEMORY.md · 채팅 · 파일이 하나로 묶여 있다는 점입니다. v3 Memory는 그 사실을 매 화면에서 가시화합니다.

  • ① Eyebrow system — v2의 파란 막대 + 모노 라벨을 시그너처화. 메모리(코덱스 그린) · 에이전트(클로드 코랄) · 파일(그레이) · 프로젝트(브랜드 블루) 4종.
  • ② Context Pin (composer 위 1줄) — 현재 ARIS가 보고 있는 컨텍스트를 chip 묶음으로. PROJ aris-web · MEM 워크플로우 규칙 · AGENT Claude · @ChatInterface.tsx.
  • ③ Memory reference (인라인) — 메시지 본문에서 메모리를 가리키는 단어 아래에 1px 코덱스 그린 점선. 호버 시 popover로 메모리 원문 + 마지막 수정 시각.
  • ④ Active Context 사이드 패널 — 채팅·프로젝트 화면 우측에 항상 노출되는 사이드. 메모리·에이전트·프로젝트 메타 + 첨부 파일을 그룹별로.
  • ⑤ Memory ribbon — 세션 도중 MEMORY.md / AGENTS.md가 업데이트되면 해당 카드 우상단에 잠시 리본 등장. 사용자가 그 변경을 수동으로 인지.
  • ⑥ Project doc 시그너처화 — v2 proj-doc 컴포넌트(opacity gradient prose preview)를 그대로 쓰되, eyebrow를 4종 시스템에 맞춰 컬러 분기.
원칙

컨텍스트는 읽기 가능해야 함. Context Pin은 클릭하지 않아도 한 줄로 읽힘. Active Context는 항상 보이되 default-collapsed 그룹으로 시작 — 정보 밀도와 가독성의 균형.

03 · Project · contextual

프로젝트 화면이 컨텍스트를 들고 있다.

v2의 Project 상세 뷰를 그대로 두고, 우측에 Active Context 사이드 패널을 추가했습니다. AGENTS.md · MEMORY.md preview는 v2 proj-doc을 그대로 사용하지만 eyebrow 색만 새 시스템으로.

04 · Composer · context pin

메시지를 보내기 전, 컨텍스트가 보인다.

채팅·Ask ARIS의 입력창 위에 Context Pin이 한 줄로 들어갑니다. ARIS가 이 메시지에 어떤 메모리·에이전트·파일을 함께 보낼지 사용자가 클릭 한 번 없이 인지할 수 있습니다.

06 · Risk

정보 밀도와 가독성의 균형.

① Active Context는 default-collapsed 그룹. 각 그룹 헤더에 항목 수만 보이고, 클릭해야 펼침. 한 화면에 chip 무더기가 보이지 않게.

② Context Pin은 한 줄. 5개 이상 chip 시 가로 스크롤 없이 +N more로 축약. 클릭 시 모달.

③ Memory ref는 명시적 인용에만. ARIS가 추론으로만 메모리를 참조한 경우는 표시하지 않음. 잘못된 신호가 늘어나면 신뢰가 떨어짐.

④ Eyebrow 컬러 4종으로 한정. 추가 의미축이 필요해도 --text-tertiary 그레이 막대로 통합. 확장하면 식별성 떨어짐.

결론

v2가 만든 작은 시그너처 컴포넌트(.proj-doc__eyebrow) 하나를 ARIS 전체의 메모리 정체성으로 격상한다. 새 토큰 0개, 새 색상 0개. 컨텍스트가 보이는 것이 ARIS만의 차별점.
