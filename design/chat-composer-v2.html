Option A의 Focused Minimal을 베이스로, 상단에 Agent Plan Terminal 세 모드를 전환하는 pill 토글을 얹었습니다. 선택된 모드에 따라 focus ring, Send 버튼 컬러, placeholder, 그리고 Terminal 모드에서는 textarea 글꼴까지 함께 변합니다. 모드가 곧 색이고, 색이 곧 의도를 보여줍니다.

05 · Model selector

Model과 Effort, 두 축만 남긴다.

팝오버는 Model 선택과 Reasoning effort 조절 두 가지에만 집중합니다. 상단 우측 context chip을 클릭하면 provider별 모델 목록이 위로 올라오고, 선택된 모델의 effort는 하단에 고정된 3-card 세그먼트로 항상 보입니다. ARIS는 Claude Code · Codex · Gemini 등 CLI 툴을 경유해 실행되므로 직접 API 요금 표시는 하지 않습니다. Terminal 모드에서는 모델 대신 PM2 프로세스 선택기로 전환됩니다.

10 · Rationale

왜 이 선택이었나.

A

왜 3모드? Ask는 왜 뺐나?

ARIS는 "에이전트가 실제로 일하는" 제품. 순수 Q&A는 Agent의 특수 케이스로 충분합니다. 4개로 늘리면 선택 부담만 증가.

B

왜 상단 segmented pill?

모드는 지금 무엇을 하는지의 1차 정보입니다. 가장 먼저 눈에 닿아야 하므로 top-left. 3개만 있어서 segmented pill이 가장 조밀하고 정확.

C

왜 색을 CSS variable로?

모드 전환 시 단 하나의 selector(.comp-v2--plan)로 border · focus ring · send · placeholder 등 5곳의 색이 동시에 바뀝니다. 유지보수 비용 최소.

D

왜 Terminal에서만 글꼴 변경?

Terminal은 실제 쉘 문법을 치는 곳입니다. monospace는 정확성을 높이고, 시각적으로 "이건 커맨드다"를 명확히 알립니다. 다른 모드에서는 한국어 가독성 우선.

E

왜 Send 라벨이 모드별로?

"Send"는 범용이고 "Plan/Execute"는 구체적입니다. 버튼 라벨이 동작을 정확히 예고하면 사용자는 결과를 예측할 수 있습니다. 실수 방지의 마지막 관문.

F

왜 모바일에도 동일한 토글?

모드는 의도의 단위이지 데스크톱 기능이 아닙니다. 모바일에서도 같은 3모드를 제공해야 사용자의 mental model이 일관됩니다.
