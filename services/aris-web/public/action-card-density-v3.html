E. 구현 규칙

시안 적용 시 따라야 할 행동 규칙

명령어 파싱

  • Agent-level 도구 (Read·Write·Edit·Glob·Grep·WebFetch·WebSearch·Task·TodoWrite·Think): 도구 이름을 그대로 표시.
  • Bash: 첫 토큰 추출 — 선행 env var/sudo/cd/time/env 건너뛰기. &&/||/; 의 첫 segment의 첫 명령. 파이프(|) 첫 명령 사용 + meta에 +N piped.
  • 매핑 미스 → cmd 톤으로 fallback. 명령어 텍스트는 그대로 노출.

파일 칩 (file-chip)

  • 인자에서 파일/디렉터리 경로로 보이는 토큰(./, / 포함, 또는 확장자 매칭)을 .file-chip 으로 wrap.
  • 아이콘은 folder(끝이 /) · code(.tsx/.ts/.js/.py 등) · config(.json/.yml/.toml/.env) · shell(.sh) 으로 분기.
  • chip 클릭 = 워크스페이스 파일 오픈 (dispatchWorkspaceFileOpen). hover 시 full path tooltip.

신택스 토큰

  • cmd-tok-cmd — 첫 명령 토큰 (primary 색)
  • cmd-tok-flag — --flag / -f (tertiary)
  • cmd-tok-str — "..." · '...' (success-fg green)
  • cmd-tok-op — | && || > < (b-600 blue)
  • cmd-tok-num — 숫자 (warning-fg amber)

모드 결정

  • 전역 기본값 — 4-way 토글(자동/확장/기본/최소).
  • 카드별 override — 어떤 모드에서도 클릭하면 일시적으로 확장. 다시 클릭하면 닫힘.

자동 모드

  • 실행 중 → 확장형
  • 가장 최근 완료 1 → 기본형
  • 그 이전 → 최소형 (stack 그룹화)
  • 에러 → 거리 무관 기본형 이상

최소형 stack

  • 사이에 텍스트가 없는 연속 액션은 자동으로 한 row stack에 묶임.
  • 배지에 명령어 짧은 이름 (Bash 첫 토큰 또는 agent 도구 이름).
  • tone color로 명령어 카테고리, is-running·is-error로 상태.
  • row 우측에 "N actions · 상태 요약" 라벨.
  • hover popover에 파일 칩 + 신택스 highlighting이 포함된 명령어 미리보기.
