D. 구현 규칙

시안 적용 시 따라야 할 행동 규칙

모드 결정

  • 전역 기본값 — 채팅 헤더 또는 설정의 4-way 토글(자동/확장/기본/최소)이 모든 카드의 기본 밀도를 결정.
  • 카드별 override — 어떤 모드에서도 카드/원형 아이콘을 클릭하면 그 카드만 일시적으로 확장된다. 다시 클릭하면 닫힘.

자동 모드 규칙

  • 실행 중 → 확장형
  • 가장 최근 완료 1개 → 기본형
  • 그 이전 → 최소형 (stack 그룹화 적용)
  • 에러 결과는 거리 무관 기본형 이상 유지 — stack에 묻히지 않게.

최소형 stack

  • 사이에 텍스트(에이전트/유저 메시지)가 없는 연속 액션들은 자동으로 한 row의 stack으로 묶임.
  • 원형 아이콘은 22px · -8px margin 으로 살짝 겹침.
  • 색상은 상태(success/running/error/idle), 흰색 SVG는 tool 종류.
  • row 우측에 "N actions · 요약" 라벨.

시각 상태 표식

  • 실행 중 — 파란 펄스 ring
  • 실패(exit ≠ 0) — 우상단 빨간 corner 점
  • thinking — 회색 idle 톤 (success/error 색 차지하지 않음)
